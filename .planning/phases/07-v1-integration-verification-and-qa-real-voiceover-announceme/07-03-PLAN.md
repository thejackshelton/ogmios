<objective>
Reshape the v1 SDK surface per CONTEXT.md § "API reshape question":

1. Add `end(): Promise<void>` to ScreenReaderHandle as an alias for `stop()`
   (keep stop as back-compat). Name symmetry with `start()` matches user
   intuition.
2. Add top-level `voiceOver.start(opts)` / `voiceOver.end()` that manage a
   **process-singleton** default handle. `voiceOver(opts)` (the existing
   factory form) remains for users who want explicit handle ownership.
3. Verify `reset()` is genuinely cheap — it clears the ring buffer + sends
   one AppleScript cursor-reset down the existing OsascriptShell, does NOT
   respawn osascript or restart VO. Add a test that asserts both invariants.

Purpose: CONTEXT.md Decision D-03 locks this reshape. Current Phase 3+4 shape
requires users to thread a handle through beforeAll/afterAll; the singleton
form makes the common "one VO session per test file" case one line:
`beforeAll(() => voiceOver.start())` / `afterAll(() => voiceOver.end())`.
Between tests, cheap `handle.reset()` keeps state clean.

Output: Updated TS SDK surface, updated @shoki/vitest re-exports, new tests
proving each invariant, and docs updated to show the preferred pattern.
</objective>

     // Process-singleton for the convenience API.
     let singleton: ScreenReaderHandle | null = null;
     let singletonRefcount = 0;

     async function startSingleton(opts: VoiceOverOptions = {}): Promise<ScreenReaderHandle> {
       if (process.platform !== 'darwin') {
         throw new VoiceOverUnsupportedPlatformError(process.platform);
       }
       if (singleton === null) {
         singleton = createDriverHandle({ driverName: 'voiceover', logBufferSize: opts.logBufferSize });
         await singleton.start();
       }
       singletonRefcount += 1;
       return singleton;
     }

     async function endSingleton(): Promise<void> {
       if (singleton === null) return; // no-op
       singletonRefcount = Math.max(0, singletonRefcount - 1);
       if (singletonRefcount === 0) {
         const h = singleton;
         singleton = null;
         await h.end();
         await h.deinit();
       }
     }

     // Attach to the factory fn — TS supports function-object merging.
     voiceOver.start = startSingleton;
     voiceOver.end = endSingleton;
