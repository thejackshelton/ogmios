Zig 0.16 std.process.Child reference (do NOT assume — verify against docs if in doubt):
- `std.process.Child.init(argv, allocator)` → initializes
- `.stdin_behavior = .Pipe`, `.stdout_behavior = .Pipe`, `.stderr_behavior = .Pipe`
- `.spawn()` starts it; `.stdin`, `.stdout`, `.stderr` are `?std.fs.File`
- Kill via `.kill()`, wait via `.wait()`.
- For a line-bounded read with timeout, use `std.fs.File.reader()` + a
  deadline loop + `std.posix.poll` or a small helper thread; simplest on
  0.16 is a dedicated reader thread pushing lines into a mutex-guarded
  queue with a cond-var.

A previous `MockChildProcess` in zig/test/voiceover_applescript_test.zig
already encodes the write-line / read-line / sentinel protocol — study it
before writing RealChildProcess so we match the contract exactly.
