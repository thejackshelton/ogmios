<objective>
Strip `.app` bundles out of the npm binding tarballs. Bindings become pure native-addon packages (`shoki.node` + README + LICENSE), shrinking from ~10MB+ (with two .app bundles) to ~2MB. Rewire `.github/actions/build-helper-app/action.yml` and `.github/workflows/release.yml` so the helper-build CI step no longer copies .apps into the binding directories.

Purpose: This plan and Plan 03 are complementary halves of the distribution split — Plan 03 ships the .apps via GitHub Releases, this plan removes them from npm. Together they enable Plan 02's `shoki setup` flow: SDK installs cheaply via npm, .apps download on demand from a separate release channel.

Output: Smaller binding tarballs, simpler SDK release pipeline, no duplication between npm and GitHub Releases.
</objective>
