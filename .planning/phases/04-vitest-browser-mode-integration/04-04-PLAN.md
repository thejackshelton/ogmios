```ts
import '@shoki/matchers/setup';
// expect(log: ShokiEvent[] | WireShokiEvent[]).toHaveAnnounced({ role, name, source?, interrupt? })

```

Important: `@shoki/matchers` matchers accept `ShokiEvent[]` shape (with `tsNanos: bigint`). The Vitest plugin's BrowserCommands return `WireShokiEvent[]` (with `tsMs: number`). The matchers check structural fields (`role`, `name`, `source`, `phrase`, `flags`) — all of which are IDENTICAL in both shapes. The matchers do NOT care about the timestamp field; they iterate the array and match `role`/`name`/etc. So `expect(wireLog).toHaveAnnounced(...)` works correctly. Document this in the README.

<test_gating>
## Why and how to gate the integration test

Phase 3's integration tests gate behind `SHOKI_INTEGRATION=1` and Phase 3's native build tests gate behind `SHOKI_NATIVE_BUILT=1`. The Vitest example's canonical test does both:

- Without `SHOKI_INTEGRATION=1`: skip the test with a clear message ("Set SHOKI_INTEGRATION=1 and run on macOS with VoiceOver set up to exercise this test.").
- On darwin with `SHOKI_INTEGRATION=1`: run the full test — boot real VO via the plugin, click the button, assert on the log.
- On non-darwin: skip regardless (VoiceOver doesn't exist).

This keeps the example installable + typecheckable everywhere; only the VO-dependent assertion needs a real mac.
</test_gating>

    ```tsx
        afterAll(async () => {
          if (session) {
            await session.stop();
            session = undefined;
          }
        });

        it.skipIf(!runVoTest)(
          'announces the Submit button on click and shows the Form submitted status',
          async () => {
            if (!session) throw new Error('session not initialized');

            render(<App />);
            // Wait for VO to settle after the render.
            await session.awaitStable({ quietMs: 500, timeoutMs: 10_000 });
            await session.reset();

            // Focus + click the Submit button via Playwright's built-in commands.
            const submit = page.getByRole('button', { name: 'Submit' });
            await submit.click();

            // Wait for both the focus-move announcement (if any) and the live-region announcement to settle.
            const log = await session.awaitStable({ quietMs: 500, timeoutMs: 10_000 });

            // Primary assertion: the Submit button was announced with its role + accessible name.
            expect(log).toHaveAnnounced({ role: 'button', name: 'Submit' });

            // Secondary assertion: the live region announced the success text.
            // Some VO configurations/rates may coalesce; allow either an explicit "Form submitted" phrase
            // OR a status role announcement with the matching text.
            expect(log).toHaveAnnouncedText(/Form submitted/i);
          },
          30_000,
        );
      });
    });
    ```

    ## What's happening under the hood
