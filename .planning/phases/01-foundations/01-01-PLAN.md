<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → user machine | User installs untrusted packages; npm is the trust root |
| Monorepo root → sub-packages | Workspace tool resolves cross-package deps; malicious workspace package could hijack via lifecycle scripts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | npm postinstall lifecycle | mitigate | No `postinstall`/`preinstall` scripts anywhere — binaries ship IN platform packages, not downloaded at install. Enforced by `optionalDependencies` pattern (per PITFALLS.md Pitfall 12 + enterprise `--ignore-scripts` compatibility). |
| T-01-02 | Information Disclosure | `.env`, credentials in repo | mitigate | `.gitignore` lists `.env`, `.env.local`, editor folders. No secrets committed in scaffolding. |
| T-01-03 | Elevation of Privilege | Unverified workspace dep resolution | accept | pnpm 10 resolves workspace:* only to local packages by design; no external registry lookups for workspace refs. |
</threat_model>
