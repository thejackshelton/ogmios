<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zig ↔ OS process table | `pgrep`/`pkill` boundary |
| Zig signal handler ↔ main Zig code | Signal handler must be async-signal-safe |
| Zig ↔ VoiceOver via AppleScript | Boot/quit commands cross here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-20 | Denial of service | signal handler blocks on heap allocation | mitigate | crashRestore uses pre-allocated snapshot; subprocess spawns use fork+exec which is signal-safe |
| T-03-21 | Elevation of privilege | forceKillVO running as root kills root-VO | accept | Users run tests as themselves; pkill requires same UID. Cross-user kills would require setuid. |
| T-03-22 | Spoofing | another process named "VoiceOver" catches the pgrep | accept | `pgrep -x` matches exact name; system VoiceOver is at a fixed path; no practical spoofing surface |
| T-03-23 | Tampering | malicious signal delivery causes premature plist restore | accept | Restoring to user's prior state is benign; worst case is a false early stop |
| T-03-24 | Repudiation | VO killed silently without audit trail | accept | std.log.warn on force-kill path; user can grep logs |
| T-03-25 | Information disclosure | pgrep output reveals user's running processes | accept | Process list is already world-readable on macOS |
</threat_model>
