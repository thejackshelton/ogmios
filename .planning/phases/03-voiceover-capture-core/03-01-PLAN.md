    2. `pub const PlistSnapshot = struct { allocator, domain: []const u8 (owned), entries: [9]PlistValue, pub fn deinit(self) void }`. `deinit` frees any owned `.string` values and the domain.

    3. `pub fn snapshotSettings(allocator, runner: SubprocessRunner, domain: []const u8) !PlistSnapshot` — for each of the 9 keys in `keyCatalog()`, spawn `defaults read <domain> <key>`:
       - exit 0 + stdout parses as integer → `.integer = parsed`
       - exit 0 + stdout is "1" or "0" for a boolean-typed key → `.boolean = (stdout[0] == '1')`
       - exit 0 + stdout is other → `.string = <owned copy>`
       - exit != 0 AND stderr contains "does not exist" → `.missing = {}`
       - any other non-zero exit → return error.DefaultsReadFailed

       Store each result in `entries[i]`. Own the domain and any string values via `allocator.dupe`.

    4. `pub fn configureSettings(allocator, runner, domain, opts: InitOptions) !void` — for each of the 9 keys:
       - If the key is "mute" (index 1) and `opts.mute == false`: skip.
       - If the key is "speech rate" (index 2): write `opts.speech_rate` instead of the catalog default.
       - Else: write `catalog[i].shoki_default`.
       - Command shape: `defaults write <domain> <key> -<type> <value>` where `<type>` is `bool`/`int`/`string`. Use argv array — never string concat.
       - bool values: write "true" or "false" as the argv.

    5. `pub fn restoreSettings(allocator, runner, snapshot: *const PlistSnapshot) !void` — for each snapshot entry:
       - `.missing`: spawn `defaults delete <domain> <key>`. exit != 0 AND stderr contains "does not exist" is OK (already gone); any other non-zero exit → error.DefaultsRestoreFailed.
       - `.boolean = v`: `defaults write <domain> <key> -bool <v>`
       - `.integer = v`: `defaults write <domain> <key> -int <v>`
       - `.string = v`: `defaults write <domain> <key> -string <v>` (argv array; v may contain spaces and must not be shell-escaped)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zig ↔ `defaults` CLI subprocess | Shoki-controlled argv crosses here; no untrusted input reaches the CLI |
| User's plist file ↔ Shoki | Read/write crosses SIP-respected filesystem ACL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `defaults write` argv | mitigate | All subprocess calls use argv arrays; no string interpolation of user/opt-derived data |
| T-03-02 | Denial of service | `snapshotSettings` | accept | 9 subprocess spawns per start; ~100ms total. Acceptable for a test-start hook. |
| T-03-03 | Repudiation | `restoreSettings` failure on crash | mitigate (partial) | Plan 03 (lifecycle hooks) completes the mitigation — this plan only ensures `restoreSettings` is idempotent and safe to call from a crash handler |
| T-03-04 | Information disclosure | string values in snapshot (e.g. voice name) | accept | Voice names are not secrets; plist already world-readable |
| T-03-05 | Spoofing | running `defaults` CLI at non-Apple path | accept | Shoki invokes `defaults` as bare name; `PATH` lookup is the platform convention |
</threat_model>
