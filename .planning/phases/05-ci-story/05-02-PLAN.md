# Phase 5 Plan 02 — shoki/setup-action + kill-background-apps.sh

## Objective

Ship the reusable composite action that prepares any macOS runner for a
shoki test, and the kill-background-apps.sh script invoked both
pre-job and post-job.

## Tasks

1. `.github/actions/setup/action.yml` — composite action with:
   - `topology` input (tart / cirrus / getmac / github-hosted / auto)
   - `macos-version` input
   - `helper-path` input
   - `skip-doctor` input (for bootstraps before shoki CLI is installed)
   - Auto-detect via CIRRUS_CI / GETMAC / SHOKI_TART / /etc/shoki-image
   - Per-topology steps (no-op on tart because image is pre-baked)
   - Final `shoki doctor --json --quiet` gate
2. `.github/actions/setup/kill-background-apps.sh` — quit loop for
   Slack/Discord/Teams/Mail/Calendar/Notes/Messages/Music/Spotify/FaceTime,
   plus Notification Center launchctl unload. Best-effort, always exits 0.

## Verification

- `action.yml` is valid GH Actions composite syntax
- kill script is chmod +x
- Pre-job step runs kill script (wired inside action.yml)
- Post-job hook is documented; each reference workflow has `if: always()`
  final step invoking the same script

## Requirements coverage

- CI-03: setup-action works on all 4 topologies
- CI-05: pre/post-job kill hooks
