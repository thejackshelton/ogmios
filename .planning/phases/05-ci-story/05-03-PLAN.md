# Phase 5 Plan 03 — Reference workflows + parity gate + publish pipeline

## Objective

Ship the four reference workflows demonstrating `shoki/setup-action` per
topology, the Phase 5 parity gate, and the tag-triggered tart image
publish pipeline.

## Tasks

1. `.github/workflows/examples/shoki-tart-selfhosted.yml` — self-hosted
   tart, SHOKI_TART=1, tart topology explicit.
2. `.github/workflows/examples/shoki-cirrus-runners.yml` — Cirrus
   Runners cloud, cirrus topology.
3. `.github/workflows/examples/shoki-getmac.yml` — GetMac managed,
   getmac topology, GETMAC=1.
4. `.github/workflows/examples/shoki-github-hosted.yml` — stock
   macos-latest, github-hosted topology.
5. `.github/workflows/phase-5-parity.yml` — matrix job on all 4
   topologies running `examples/vitest-browser-react`. Final `parity`
   synthesis job for branch protection. `if: github.event_name == 'push'`
   gates on tart and getmac (fork PRs don't have self-hosted access).
6. `.github/workflows/tart-publish.yml` — canonical publish workflow.
   Matrix on macOS version; per-cell Packer build + smoke test + tart
   push to ghcr.io. `@full` opt-in via workflow_dispatch input.
7. `infra/tart/publish.yml` — pointer file (GitHub only discovers
   workflows under .github/workflows).
8. Extend `release.yml` + `ci.yml` to call setup-action on macOS jobs
   with `skip-doctor: true` (shoki CLI isn't built yet at those points).

## Verification

- All 7 workflow files parse as valid GH Actions
- Parity matrix is 4 cells
- tart publish has per-version smoke test (ssh + marker read)
- Reference workflows all reference the same shared action
- `examples/vitest-browser-react/package.json` unchanged (we're only
  running it, not modifying it)

## Requirements coverage

- CI-04: reference workflows exist for each topology
- CI-06: parity matrix exercises all 4 on the same example

## Deviations

- The instruction mentions `infra/tart/publish.yml` as the publish
  workflow but GitHub Actions only picks up workflows in
  `.github/workflows/`. Canonical file is
  `.github/workflows/tart-publish.yml`; `infra/tart/publish.yml` is kept
  as a human-navigable pointer.
