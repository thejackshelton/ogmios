# Quick Task 260417-wl1 — Style VitePress docs with ear-image palette

**Goal:** Restyle the VitePress docs site around the Shoki ear logo palette (#092036 navy background, #fc8277 coral accent, white text) and swap the hero from `/logo.svg` to a PNG of the ear illustration. Force dark mode only.

## Scope

- Download the ear logo (supplied Discord CDN URL) to `docs/public/ear.png`, normalize to real PNG, resize to 800×800.
- Create `docs/.vitepress/theme/` with:
  - `index.ts` extending VitePress's `DefaultTheme` and importing `custom.css`
  - `custom.css` overriding VitePress CSS vars (`--vp-c-bg*`, `--vp-c-brand-*`, `--vp-c-text-*`, hero, buttons, code, search) at both `:root` and `.dark`
  - `env.d.ts` declaring the CSS side-effect import so TS is happy
- Update `docs/.vitepress/config.ts`:
  - `appearance: "dark"` → `"force-dark"` (no light mode)
  - `theme-color` meta: `#646cff` → `#fc8277`
- Update `docs/index.md` hero image src from `/logo.svg` → `/ear.png` (with a descriptive alt).

## Tasks

1. **Image** — `curl` → `sips` convert WebP→PNG → resize to 800×800.
2. **Theme** — write `theme/index.ts`, `theme/custom.css`, `theme/env.d.ts`.
3. **Config + index** — edit `config.ts` (appearance + theme-color) and `index.md` (hero src).
4. **Verify** — `pnpm build` in `docs/` completes cleanly.

## Verify / Done
