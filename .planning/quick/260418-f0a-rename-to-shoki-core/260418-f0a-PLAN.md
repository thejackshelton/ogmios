<objective>
Mechanical rename of the published npm package name: `shoki` → `@shoki/core`. The `@shoki` org scope is exempt from npm's anti-typosquatting filter (which currently blocks unscoped `shoki` as too similar to `shiki`). This unblocks `npm publish` today — no appeal required.

**Critical distinctions:**
- NPM **package name** changes: `shoki` → `@shoki/core`
- CLI **bin name** stays `shoki` — users still type `shoki doctor`, `shoki setup`, `npx shoki setup`
- Project **name** "Shoki" (proper noun in prose, headings) stays everywhere
- GitHub **repo URL** stays `github.com/shoki/shoki`
- `compatibleAppVersion` field stays (app-version contract, unrelated)

This reverses (to a new scoped name) what Phase 10.1 did when it renamed `@shoki/sdk` → `shoki`. The subpath layout established by Phase 10.1 stays intact — only the root name gains an `@shoki/core` scope.

Purpose: Ship v0.1.0 to npm today without waiting 1-7 days for an npm appeal.

Output: A clean green build with the new scoped name everywhere in code and docs, ready for `npm publish --access public`.
